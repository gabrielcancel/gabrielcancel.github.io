Welcome to git tutorial

Setup

Download installer for Windows and Mac

Windows Setup

Specific option during installation :

  • Use Git and optional Unix tools form the Windows Command Prompt
  • Other parameter to set leave default

Linux Setup

apt-get install git

Mac Setup

  • Open the dmg file
  • Install dpkg file

Check the installation

Open a terminal and write

git help

or

git --help

Using git

Init git

Create folder with

mkdir folder_name

Go to this folder

cd folder_name

And init git on this folder

git init

Start with git

Create a file

touch README.md

View file status

git status

To track the file with git, use

git add README.md

If you watch again the file status, your file is tracked. You can now commit your modification

git commit -m "your comment"

Tips

To add a file and commit it in the same time, use

git commit -a -m “your comment”

To track all file, use

git add --all

Ignore file In some cases you want to ignore files. To do this, create a .gitignore

touch .gitignore

Edited this file by putting the filenames to ignore And commit this file

Watch your commit history

git log

You can specify some parameters
For example, if you want the last two commit, use

git log -n 2

The list of commits is very complete, to make it more succinct, use

git log --oneline

If you want specify a file, use

git log -p filename

Watch differences

If you want look at the differences between your actual file and file status when you commit

git diff

To go back on a commit

git checkout commitID filename

This gives us back the changes that were to the file in question at the time of the specified commit. The file is then modified and we can commit it.

To return to the current state

git checkout master filename

Revert

If we want to undo what you did at a particular commit, use

git revert commitID

Branch

Create a new branch

git branch branchname

To change branches

git checkout branchname

Merge branch

To merge branches, go to the destination branch For example go to master branch

git checkout master

And now merge yout branch

git merge branchnametomerge

You can now delete the branch that has just been merged

git branch -d branchname

Link our git to a GitHub repository

Perform all the initialization steps and create the repo on GitHub, then remote

git remote add origin repoaddress

To send all local files to the repository

git push origin —all

Congratulations you have just started using versioning
